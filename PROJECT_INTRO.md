# MedRAG — 全科医生辅助诊断系统

---

## 一句话概述

> 基于 **MedRAG 医学知识检索增强** + **DeepSeek LLM 临床思维链**的未分化疾病智能辅助诊断平台，帮助基层医生快速完成从症状采集到鉴别诊断的全流程推理。

---

## 项目背景

我国基层医疗面临两大痛点：**全科医生数量不足**和**未分化疾病诊断困难**。患者在就诊初期往往只能描述模糊症状（如「头晕、乏力」），医生需要在海量疾病知识中快速锁定可能病因。

MedRAG 系统借鉴学术界 **MedRAG（Medical Retrieval-Augmented Generation）** 思想，融合知识图谱、临床指南与 LLM 推理能力，构建了一条「症状输入 → 知识检索 → 结构追问 → 鉴别诊断 → 治疗方案」的完整诊断链路。

---

## 核心功能

| 功能模块 | 说明 |
|----------|------|
| 🔍 **多症状分析** | 自然语言输入症状，LLM 自动拆解为结构化医学特征 |
| 📚 **知识增强检索** | MedRAG 融合 DDXPlus 知识图谱，覆盖呼吸/心血管/消化/神经/骨骼五大系统 |
| 🤖 **临床思维链** | 调用 DeepSeek-chat API，模拟医生逐步推理、动态追问 |
| 💬 **智能多轮追问** | 3 轮结构化追问，每轮根据回答动态调整诊断方向 |
| 📊 **置信度更新** | 每轮追问后实时更新疾病匹配置信度 |
| 📋 **诊断报告** | 输出初步诊断、鉴别诊断、诊断依据、建议检查、治疗方案、注意事项 |
| 🌐 **Web 界面** | Vue 3 响应式前端，适配 PC 和移动端 |
| 🐳 **Docker 部署** | docker-compose 一键启动，生产环境就绪 |

## 医学知识库覆盖

| 系统 | 代表疾病 |
|------|----------|
| 🫁 呼吸系统 | 上呼吸道感染、流感、急性支气管炎 |
| 🫀 心血管系统 | 心绞痛、原发性高血压 |
| 🍽️ 消化系统 | 急性胃炎、功能性消化不良、胃食管反流 |
| 🧠 神经系统 | 偏头痛、紧张性头痛 |
| 🦴 肌肉骨骼系统 | 腰椎间盘突出、骨关节炎、颈椎病 |

> 知识库基于 DDXPlus 数据集构建，可通过 MedRAG 框架动态扩展。

## 诊断流程

```
患者输入症状
    │
    ▼
┌─────────────┐
│ 第 1 步      │ 症状分析：LLM 拆解关键症状，匹配知识库
│ 症状分析     │ 输出：可能疾病列表 + 初始置信度
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 第 2 步      │ 智能追问：LLM 生成针对性补问（最多 3 轮）
│ 动态追问     │ 每轮根据回答缩小诊断范围、更新置信度
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 第 3 步      │ 诊断报告：综合症状、追问、知识库，LLM 生成
│ 诊断报告     │ 含初步诊断、鉴别诊断、检查建议、治疗方案
└─────────────┘
```

## 快速开始

```bash
# 1. 环境配置
cp .env.example .env
# 编辑 .env 填入 DEEPSEEK_API_KEY

# 2. Docker 一键启动
docker-compose up -d

# 3. 访问
# 前端: http://localhost
# API文档: http://localhost:8000/docs
```

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/analyze` | POST | 症状分析 → 初步诊断 |
| `/api/diagnose` | POST | 最终诊断报告生成 |
